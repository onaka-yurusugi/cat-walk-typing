- No routing library, no state management library — single-page app with local React state

## Architecture

This is a browser-based typing game ("黒猫バシバシタイピング") where players type pseudo-random strings generated by simulating a cat walking across a QWERTY keyboard.

### Game flow (3 states)

`title` → `playing` (60s timer) → `result` — managed by `GameState` union type in `useGame`.

### Key modules

- **`src/hooks/useGame.ts`** — Core game logic hook: timer, keydown handling, scoring (base 10pts + combo bonus), and paw-strike animation state. All game state lives here.
- **`src/lib/generateCatString.ts`** — "Cat walk" string generator: simulates movement across adjacent QWERTY keys (80% move / 20% stay), with a 30% chance "sleep event" that repeats one key 15–39 times. Also exports the `KEYBOARD` layout constant used by the virtual keyboard UI.
- **`src/components/CatPaw.tsx`** — SVG component for the cat paw strike animation.
- **`src/App.tsx`** — Single component rendering all 3 game screens and the virtual keyboard. Inline `<style>` defines the `@keyframes strike` animation.
